Final Report

Our original goal for this project was to design a solution that would improve wayfinding throughout the Mattress Factory and to enhance visitor experience by presenting more information about the pieces and how to interact with them.

This report describes are final user testing sessions and provides documentation for client maintenance of the application

User testing

April 24, 2012

We tested the new expandable navigation to see if users could figure out the location as well as identifying the other buildings. Most of the users were able to identify their current location as well as other buildings. Few users were able to identify where they were in the building. Because users were not at the Mattress factory we attributed this problem to their context.

May 5, 2012

Once we got most of the development done, we wanted to test the hi-fi prototype to get any last minute user input in and make the changes before the project was finished. We used our phones to test people on each floor. We handed them our phones and asked the users to walk around on the same floor to test it out. When they were done with the floor, we asked if they had any comments about the application. We also prompted them to tell us whether or not they thought it was useful, how the navigation was, and which functionalities they thought were useful.

We tested around 4-5 people on each floor. We found out that most people thought it was a really neat concept and found it useful. Everyone really liked the navigation concept that we had on the main screen and did not have any troubles with it.

On the details page, most people clicked the general info button but also clicked the temporary label on the piece.

However, because the application was not in its final stage of development, there were many bugs that arose. A recurring problem was that the page did not load fully at times, and it would only show the header bar with a white screen. This was mostly caused by pressing the back button that we implemented. Refreshing the page usually put the page back to being functional.

Nobody tried out the tagging because most people thought it was not functional due to their previous experience with all the bugs. When someone went to a new page that did not have any tags yet, this also caused problems because it shows an empty box with a tag button. The users were confused with what the tag button did.

Overall, the user testing was very helpful and we continued to iterate our design based on these results.
